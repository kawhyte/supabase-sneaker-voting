feat(Phase 7.2): Create comprehensive testing guides for edge function deployment

This commit completes the testing infrastructure for Phase 7.2 (Price Monitoring System).
All testing documentation and deployment scripts are now ready for the user.

## Features Added

### Testing Documentation
- DEPLOY_AND_TEST_STEPS.md: Quick 3-minute deployment & test guide with copy-paste curl commands
- EDGE_FUNCTION_TESTING_GUIDE.md: Comprehensive 200+ line testing guide covering:
  * Local testing with Supabase CLI
  * Production deployment steps
  * 3+ testing scenarios (basic, price detection, error handling)
  * Troubleshooting guide with common issues & solutions
  * Cron job setup instructions
  * Monitoring & health check procedures
- TESTING_SUMMARY.md: Quick reference with success criteria checklist
- QUICKSTART_TEST.sh: Automated bash script for one-command testing
- EDGE_FUNCTION_STATUS.txt: Visual ASCII summary of deployment status

### Test Scenarios Documented
1. Basic Functionality: Verify function runs without errors
2. Price Drop Detection: Confirm price changes are detected correctly
3. Error Handling: Verify function handles failures gracefully (auto-disables after 3 failures)
4. CronJob Setup: Instructions for automatic weekly scheduling
5. Database Verification: SQL queries to verify alerts were created

### Success Criteria
- [ ] Edge function deployed: `supabase functions deploy check-prices`
- [ ] Function accessible: curl returns 200 OK with expected response
- [ ] At least 1 wishlist item exists with a valid URL
- [ ] Response shows `"checked": X` (X > 0)
- [ ] Price alerts appear in database (or checked > 0 confirms execution)
- [ ] No 401/404/500 errors
